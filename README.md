# Nexa Search Node.js

Telegram-бот для поиска свободных `@username`: генерация вариантов, проверки через Telegram Bot API, `t.me` и Fragment, история, лимиты, рефералка и админка. Оплата Premium в интерфейсе **временно отключена** (техработы); выдача Premium админ-командами сохраняется.

Кнопки автоматического забора ника здесь нет: бот только показывает найденный вариант и ссылки для ручной проверки.

## Почему Node.js + TypeScript

- Хорошо ложится на Telegram-бота: асинхронные проверки и inline-кнопки пишутся компактно.
- `node:20-alpine` деплоится просто и быстро.
- TypeScript держит код аккуратнее, чем один большой скрипт.
- Нет нативной SQLite-зависимости: хранилище JSON, поэтому сборка легче на Alpine.

## Запуск

1. Установи зависимости:

```bash
npm install
```

2. Открой `.env` и заполни `ADMIN_IDS`, `SUPPORT_USERNAME` и остальные настройки под себя.

`BOT_TOKEN` в `.env` намеренно отсутствует. Передавай его переменной окружения в панели bothost.

3. Запусти в dev-режиме:

```bash
npm run dev
```

4. Для продакшена:

```bash
npm run build
npm start
```

## Команды

- `/start` — главное меню.
- `/id` — узнать свой Telegram ID.
- `/admin` — админ-панель.
- `/premium ID 30d` — выдать Premium, доступно админам.
- `/unpremium ID` — снять Premium.
- `/ban ID`, `/unban ID` — бан и разбан.
- `/broadcast текст` — рассылка всем активным пользователям.

## Что можно настроить

Все ключевые параметры лежат в `.env`: число бесплатных запросов, ежедневное восстановление, бонус за реферала, cooldown, путь к файлу данных. Переменные `PRICE_PREMIUM_*` зарезервированы на будущее, когда магазин Premium снова откроют.
